# Usage tips — one per line. Shown at random on shell startup.
# Lines starting with # and blank lines are ignored. Opt out: DOTS_NO_TIPS=1
# Edit freely; `dots tips` lists them all.

# --- moving around ---
`z <part-of-path>` jumps to a frecent directory (zoxide). `zi` picks interactively (television).
`fcd` fuzzy-finds a directory and cd's into it. `cd -` toggles back.
`lt` shows a 2-level tree (eza); `ll` is a detailed list with icons.
WezTerm is the multiplexer: `Ctrl-Space \` splits horizontal, `Ctrl-Space -` vertical.
Ctrl-h/j/k/l moves between WezTerm panes AND Neovim splits seamlessly — no leader.
`Ctrl-Space z` zooms a pane; `Ctrl-Space w` switches workspaces.

# --- searching ---
`rg <pattern>` searches file contents fast; add `-t go` / `-t rust` to scope by language.
`fd <name>` finds files by name (respects .gitignore, skips .git).
Press Ctrl-R for fuzzy shell-history search (television) — type any fragment of an old command, then ↑/↓ + Enter to run it.
Press Ctrl-T to fuzzy-complete the current command (television) — files, branches, processes, depending on context.
`tv` opens the fuzzy finder; `tv text` greps file contents, `tv tldr` browses cheatsheets, and any list pipes in: `git branch | tv`.
`ff` fuzzy-finds a file and opens it in Neovim; `fcd` jumps into a directory (both via television).
`fsym` (or `tv symbols`) fuzzy-searches code symbols — classes/funcs/vars via universal-ctags — and opens the pick in Neovim at its line.
`proj` (or `tv projects`) picks a git repo under ~/projects and cd's in; Ctrl-S jumps straight to that repo's symbols.
In Neovim: `<space>ff` finds files, `<space>/` greps the project, `<space>fv`/`<space>sv` open the television picker.

# --- editing ---
Neovim is $EDITOR everywhere — git, kubectl, and anything else that opens an editor uses nvim.
`v <file>` or `ff` opens Neovim; `<space>e` toggles the file tree.
`c` opens Claude Code in the current directory (alias for `claude`).
`lg` opens lazygit — stage hunks, write commits, manage branches visually.
Commit subjects need a ticket key (e.g. ROG-1234) — the commit-msg hook enforces it.
`gf`/`gd` in Neovim jump to definition; `K` shows hover docs; `<space>ca` for code actions.
`<space>cr` renames a symbol project-wide; `<space>cf` formats the buffer.
Add language support in Neovim with `:LazyExtras` — toggle a row, restart.
Press Space in Neovim and wait — which-key pops a menu of every binding. No need to memorize.
`y` opens yazi: a file manager with image/video previews; quit and your shell cd's there.
Inside yazi, `z` fuzzy-jumps to any file/dir and `Z` to a frecent dir — both via television.

# --- git & hooks ---
Direct commits to main/master are blocked — branch with `git switch -c feat/ROG-1-x`.
Bypass a hook once with `git commit --no-verify`; turn all off with `dots hooks disable`.
`git lg` shows a graph log; delta renders side-by-side colored diffs automatically.
In a delta diff, `n`/`N` jumps between files/hunks (navigate is on).
Pushing runs lint + fast tests for the repo's languages; skip with `HOOKS_SKIP_TESTS=1`.
Branch as `feat/ROG-1-x` and the ROG-1 ticket is auto-inserted into your commit subject.
Pushing WIP/fixup! commits is blocked — squash first (`git rebase -i --autosquash`) or `--no-verify`.
Commits auto-rebase on pull and push auto-sets upstream — no `-u` needed.

# --- maintenance & tooling ---
`dots update` upgrades configs + every toolchain (brew, rust, swift, uv, nvim, doom).
On macOS, `dots autoupdate` upgrades CLI toolchains every 4h in the background (casks wait for `dots update`) — `status`/`log` to peek.
`dots edit <file>` edits the managed source and applies it — never edit managed dotfiles in place.
`dots readd` pulls machine-local changes back into the repo; `dots save` pushes them.
Add a secret with `dots secret-add <path>` — it's encrypted before it touches git.
`man <cmd>` is rendered with syntax highlighting via bat. `cat` is bat; `ls` is eza.
`colima start` boots the container runtime; then `docker run ...` works without Docker Desktop.
Per-language tools (goimports, gopls, dlv, clippy, pytest, mypy, …) ship preinstalled.
Add any tool with `dots packages` — it opens packages.yaml and re-runs the installers on save.
`tldr <cmd>` shows example-first usage (tealdeer) — faster than scrolling a full man page.
`just` runs project tasks from a justfile: bare `just` lists recipes, `just <name>` runs one.
`git dft` shows a structural, syntax-aware diff (difftastic); `git absorb` auto-fixups staged hunks.
`btop` is a live system monitor; `dust` = disk usage, `duf` = free space, `procs` = process search.
`sd 'old' 'new' file` is a saner find-and-replace than sed; `hyperfine 'cmd'` benchmarks it.
`ls | mail you@x.com -s "subject"` emails the pipe via Apple Mail (`--cc`, `--bcc`, `--from`, `-a [file…]` to attach, `--draft` to review first). `<to>` can be a Contacts name.
`git log -15 | fmail "Kian" "my recent commits"` — a model writes a grounded summary (words = intent, pipe = material) and opens a Mail draft. Recipients can be Contacts names; short input stays on-device, long input goes to the cloud. `--send` ships it directly.
